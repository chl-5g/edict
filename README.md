⚔️ 三省三部制 · Edict

用 1300 年前的帝国制度，重新设计 AI 多 Agent 协作架构。
不是复古，而是用“分权制衡”解决多 AI 协作的失控风险。

---

🏛️ 什么是三省三部制？

三省三部制 是一套 流程安全架构，专门解决多 AI Agent 协作时的 决策黑箱、质量失控、执行偏离、责任模糊 问题。

它将人类官僚体系中最精华的 分权制衡 思想，转化为 AI 协作的 强制流程 + 独立审计 + 步骤级审批。

```
         👑 皇帝（用户）
               │
               ▼
         📜 中书省（Gemini）   ← 宏观规划
               │
               ▼
         🔍 门下省（GPT）      ← 审核封驳
               │
               ▼
         📮 尚书省（Claude）   ← 拆解调度
               │
         ┌─────┼─────┐
         ▼     ▼     ▼
      户礼部 工兵部 吏刑部   ← 三部执行（信息/构建/质量）
         │     │     │
         └─────┼─────┘
               ▼
         ⚖️ 御史台（GPT-5.4）  ← 独立审计 + 审批门禁
               │
               ▼
         👑 皇帝
```

---

🎯 解决了什么问题？

问题 三省三部制的解法
决策黑箱 强制流程：规划→审核→拆解→执行→审计，每一步有记录
质量失控 门下省“封驳”机制，在审核层拦截错误方案
执行偏离 子任务清单需皇帝过目，每步完成后审计，通过才继续
责任模糊 每个阶段有明确负责部门，审计报告可追溯
资源滥用 尚书省拆解任务+资源预估，御史台审计成本

📋 角色与职责

三省（决策与制衡）

Agent 模型 职责
中书省 Gemini 3.1 Pro 理解需求，产出《项目总体方案》（技术栈、模块划分、验收标准）
门下省 Gemini 3.1 Pro 审核方案可行性、风险、资源；不合格则 封驳 退回
尚书省 Claude Opus 4.6 拆解总体方案为子任务，分发给三部，协调进度

三部（执行）

Agent 模型 职责 输出示例
户礼部 Claude Opus 4.6 数据层 + 文档 schema.sql, swagger.yaml, README.md
工兵部 Claude Opus 4.6 开发 + 安全编码 业务代码、安全加固、Dockerfile
吏刑部 GPT-5.4 测试 + 合规 测试报告、合规意见、缺陷列表

一台（审计）

Agent 模型 职责
御史台 Gemini 3.1 Pro 收集执行日志，生成《审计奏折》，作为 审批门禁

---

🔁 核心流程（步骤级审批）

```
1. 中书省规划 → 产出《项目总体方案》
2. 门下省审核 → 通过 / 封驳（驳回则返回1）
3. 尚书省拆解 → 生成子任务清单
4. 皇帝审阅子任务清单 → 作出批复
   ├── 【准奏】→ 进入执行阶段
   └── 【驳回】→ 任务终止或返回中书省修改方案（返回1）
5. 执行第一个子任务（户礼部/工兵部/吏刑部）
6. 子任务完成 → 保存日志 → 同步进度给三省（中书、门下、尚书）和御史台
7. 御史台审计 → 生成《审计奏折》
8. 皇帝审阅奏折 → 作出批复
   ├── 【准奏】→ 继续下一个子任务（返回5）
   └── 【驳回】→ 任务暂停，皇帝提出修改意见 → 修改意见经门下省审核、中书省修订方案 → 返回3（重新拆解）
9. 所有子任务完成后 → 御史台生成《最终审计奏折》→ 归档
```

关键设计：

· 子任务清单必须皇帝过目：确保人类对任务分解的最终确认，防止方向错误。
· 进度同步：每个子任务完成后，三省和御史台实时掌握最新进度，决策层始终知情。
· 批复仅两种：皇帝只有“准奏”或“驳回”，驳回时必须给出修改意见，确保决策清晰。
· 修改闭环：驳回的修改意见会重新进入门下审核、中书修订流程，保证问题修正后再执行。

🚀 快速开始

前置条件

· OpenClaw 已安装
· Python 3.9+
· macOS / Linux

一键安装

```bash
git clone https://github.com/cft0808/edict.git
cd edict
chmod +x install.sh && ./install.sh
```

启动

```bash
# 终端 1：数据刷新循环
bash scripts/run_loop.sh

# 终端 2：看板服务器
python3 dashboard/server.py

# 打开浏览器
open http://127.0.0.1:7891
```

详细教程请查看 Getting Started
